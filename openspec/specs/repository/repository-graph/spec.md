# Repository Graph

## Purpose
Define the normative behavior for repository graph.

## Requirements

### Requirement: packages files symbols and tests
The system MUST implement packages files symbols and tests with explicit validation and tests.

#### Scenario: Accepted behavior
- **WHEN** the capability is invoked with valid input
- **THEN** the specified behavior is deterministic and observable

### Requirement: requirement and ADR links

### Requirement: query API
The system MUST implement query api with explicit validation and tests.

## Non-goals
- Vendor-specific shortcuts in platform-neutral packages.
- Hidden mutable global state.
- Success results that conceal partial failure.
