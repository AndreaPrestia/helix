# Identifier Conventions

- Workspace: `ws_<ulid>`
- Project: `prj_<ulid>`
- Specification: `HELIX-DOM-003`
- Requirement: `HELIX-SPEC-001`
- Task: `TASK-HELIX-0001`
- Review: `rev_<ulid>`
- Event: `evt_<ulid>`
- Knowledge: `KNW-HELIX-001`
- Context: `context.helix.foundation`

Human-readable specification and requirement IDs are immutable after approval.
Runtime entities use ULIDs.
