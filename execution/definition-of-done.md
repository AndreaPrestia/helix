# Definition of Done

A change is done only when:

- proposal and design are approved;
- every normative requirement has test evidence;
- all tasks are checked;
- typecheck, lint, test, architecture test, and build pass;
- security-sensitive changes include threat analysis;
- no unapproved dependency or public API is introduced;
- verification evidence is committed;
- independent review has no unresolved blocking finding;
- the delta is merged into baseline specs and archived.
